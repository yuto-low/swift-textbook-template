# GitHub & Markdown — これだけ覚えるシート

> iOSプログラミングII 第1回配布資料

---

## ■ やることは3つだけ

この授業でGitHubを使ってやることは、たった3つです。

1. **Markdownファイルを編集して教科書を書く**
2. **保存する（コミット）**
3. **クラスメイトの教科書にコメントする（Issue）**

これ以外のGitHub機能は、この授業では使いません。

---

## ■ STEP 1：教科書テンプレートをForkする（最初に1回だけ）

「Fork」とは、他の人のリポジトリを自分のアカウントにコピーすることです。
コピーした後は自分のリポジトリになるので、自由に編集できます。

1. https://github.com にログイン
2. https://github.com/NakasenJEC/swift-textbook-template
を開く
3. 右上の「**Fork**」ボタンをクリック
4. 「**Create fork**」をクリック → 完了！

これで、教科書の全テンプレート（README、章テンプレート8つ、AI質問ログ8つ、ピアレビュー記録4つ）が自分のアカウントにコピーされます。

💡 **Forkボタンが見つからない？** ページ右上、Starボタンの隣にあります。

---

## ■ STEP 2：模範コードをダウンロードする（最初に1回だけ）

模範コードは、教員のリポジトリからZIPファイルでダウンロードします。

1. https://github.com/NakasenJEC/ios2-sample-code
を開く
2. 緑色の「**Code**」ボタンをクリック
3. 「**Download ZIP**」をクリック
4. ダウンロードされたZIPファイルを展開（ダブルクリック）
5. 中のフォルダ（`01_WebAPI_基本` など）をわかりやすい場所に保存

💡 全8章分の模範コードが一度に手に入ります。毎回ダウンロードし直す必要はありません。

---

## ■ STEP 3：ファイルを編集する

1. 自分のリポジトリ（Forkした教科書）を開く
2. 編集したいファイルをクリックして開く
3. 右上の **鉛筆アイコン** ✏️ をクリック
4. 内容を修正する
5. **「Preview」タブをクリックして見た目を確認する**（いつでも何度でも切り替えOK）
6. 問題なければ「**Commit changes**」で保存

💡 **「Commit」= 保存** と思ってOK。変更の履歴が自動で残ります。

💡 **Previewの活用：** 編集画面の上部にある「**Edit**」と「**Preview**」のタブを切り替えると、Markdownがどう表示されるかをその場で確認できます。見出しの大きさ、表の崩れ、コードブロックの色分けなどを保存前にチェックする習慣をつけましょう。

---

## ■ 新しいファイルを作る（必要になったら）

1. リポジトリのページで「**Add file**」→「**Create new file**」
2. ファイル名を入力（例：`chapters/01-webapi.md`）
   - `/` を入れるとフォルダが自動で作られる
3. 本文を書く（Markdown記法で — 次のセクション参照）
4. ページ下の「**Commit changes**」をクリックして保存

💡 テンプレートに最初から入っているファイルだけで進められるので、通常はこの操作は不要です。

## ■ Markdown記法：これだけ覚える

### 見出し

```markdown
# 大見出し（章タイトル）
## 中見出し（セクション）
### 小見出し
```

### 太字

```markdown
**このように囲むと太字になる**
```

→ **このように囲むと太字になる**

### Swiftコードを載せる

コードを書くときは、バッククォート3つで囲みます。
バッククォートは、キーボードの `Shift + @` で入力できます（日本語キーボードの場合）。

````markdown
```swift
struct ContentView: View {
    var body: some View {
        Text("Hello")
    }
}
```
````

↑ これが ↓ こう表示される：

```swift
struct ContentView: View {
    var body: some View {
        Text("Hello")
    }
}
```

### 表

```markdown
| 項目 | 説明 |
|------|------|
| async | 非同期処理 |
| await | 完了を待つ |
```

→ こう表示される：

| 項目 | 説明 |
|------|------|
| async | 非同期処理 |
| await | 完了を待つ |

### 番号つきリスト

```markdown
1. 最初にやること
2. 次にやること
3. 最後にやること
```

## ■ ピアレビュー：Issueの書き方

クラスメイトの教科書にフィードバックするときに使います。

1. 相手のリポジトリを開く
2. 上のタブで「**Issues**」をクリック
3. 「**New issue**」をクリック
4. タイトルと本文を書く
5. 「**Submit new issue**」で送信

**書き方の例：**

```
タイトル：第1章のWebAPIの説明がわかりやすかった＋質問

本文：
guard let url の説明がすごくわかりやすかったです。

1つ質問：JSONDecoderのところで「なぜdoブロックが必要か」の
説明がもう少し詳しいとさらに良いと思いました。
```

## ■ 教科書をPDFにする・製本する

完成した教科書は、PDFに変換して印刷・製本することもできます。
デザインにこだわって、世界に1冊だけの本を作りましょう。

**簡単な方法（ブラウザだけでOK）：**

1. GitHubで完成したMarkdownファイルを開く（Previewが自動表示される）
2. ブラウザのメニューから「**印刷**」（`Ctrl + P` / `Cmd + P`）
3. 出力先を「**PDFとして保存**」に変更
4. 「**保存**」をクリック → PDFファイルの完成！

💡 **もっとこだわりたい人へ：** Markdownを美しいレイアウトのPDFに変換できる無料ツールもあります（授業中に紹介します）。表紙をデザインしたり、ページ番号を入れたり、本格的な教科書に仕上げることができます。

💡 **製本サービス：** できあがったPDFは、オンラインの製本サービスを使えば1冊から本として印刷できます。自分だけのSwiftプログラミング教科書が本棚に並ぶ日を目指しましょう。

## ■ もっと知りたい人へ：git cloneでコードを取得する方法

> この授業ではDownload ZIPで十分ですが、ターミナル操作に興味がある人向けの補足です。

`git clone` は、リポジトリの内容をターミナル（コマンドライン）からPCにダウンロードするコマンドです。

**事前準備：** Macの場合、Gitは最初から入っています（XcodeのCommand Line Toolsに含まれます）。

**手順：**

1. GitHubで対象リポジトリを開く
2. 緑色の「**Code**」ボタンをクリック
3. 表示されるURLをコピー（HTTPSタブのもの）
4. ターミナルを開いて以下を実行：

```bash
cd ~/Desktop
git clone コピーしたURL
```

5. デスクトップにリポジトリ名のフォルダが作成される

💡 `git clone` は「Download ZIP」と同じく、リポジトリの中身をPCにダウンロードする方法です。違いは、gitの履歴情報も一緒にダウンロードされることです。この授業では特に必要ありませんが、将来のチーム開発では必須の操作になります。
